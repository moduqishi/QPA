页面导航结构

QPA 管理面板有 7 个功能模块，怎么组织它们？点击选择你喜欢的布局。

A

左侧导航栏（经典后台）

左侧固定侧边栏，顶部放 Logo 和 QPA 标题，下面依次是：仪表盘、账号管理、API 密钥、请求历史、用量统计、日志、系统设置。主内容区在右侧。

优点

  • 最经典的管理后台布局，用户零学习成本
  • 导航始终可见，切换页面快
  • Naive UI 的 NLayout + NMenu 原生支持

缺点

  • 侧边栏占约 200-240px 宽度，小屏幕可用空间减少
B

顶部导航栏

顶部水平导航条，放 Logo + 7 个 Tab 页签。主内容区全宽展示。导航栏固定在顶部。

优点

  • 主内容区全宽，表格和图表空间大
  • 视觉更简洁

缺点

  • 7 个 Tab 在移动端可能溢出，需要折叠菜单
  • 顶部高度有限，扩展性差
C

左侧导航 + 仪表盘首页（推荐）

和 A 类似的左侧导航，但默认首页是「仪表盘」— 一个聚合了核心指标（今日调用、token 用量、账号状态、最近请求）的概览页面。其他模块通过侧边栏进入。

优点

  • 经典布局 + 信息聚合首页，打开就能看到全貌
  • 仪表盘可放图表趋势，一目了然
  • 最适合日常运维场景

缺点

  • 仪表盘开发量稍大，但价值最高
